Javascript
Convention Example
Always put spaces around operators ( = + - * / ) var x = y + z;
Always end a simple statement with a semicolon. var values = ["Volvo", "Saab", "Fiat"];
  • Put the opening bracket at the end of the first line.
  • Use one space before the opening bracket.
  • Put the closing bracket on a new line, without leading spaces.
  • Do not end a complex statement with a semicolon.
 For Functions, loops, conditionals.
For readability, avoid lines longer than 80 characters.
use camelCase for identifier names (variables and functions).
All names start with a letter.

Scheme
Convention Example
Predicate names end in a question mark ( ? ) eq?, zero?, and string=?. The common numeric comparators =, <, >, <=, and >= are exceptions to this rule.
The names of most character, string, and vector procedures start with the prefix char "-". string-append

Identifiers may be formed with

  • the lowercase letters a through z
  • the uppercase letters A through Z
  • the digits 0 through 9
  • the characters ? ! . + - * / < = > : $ % ^ & _ ~
 hi, Hello, n, x, x3, and ?$&*!!! are all identifiers
Keywords, variables, and symbols are collectively called identifiers.
Structured forms and list constants are enclosed within parentheses (a b c) or (* (- x 2) y)
The boolean values representing true and false are written as #t and #f. Scheme conditional expressions actually treat #f as false and all other objects as true.
Vectors are written similarly to lists, except that they are preceded by #( and terminated by ) #(this is a vector of symbols)
Strings are enclosed in double quotation marks
Characters are preceded by #\ #\a
Comments may appear on any line of a Scheme program, between a semicolon ( ; ) and the end of the line
Courtesy of scheme.com, http://www.scheme.com/tspl2d/intro.html
